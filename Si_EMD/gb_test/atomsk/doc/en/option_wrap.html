Option: wrap

Syntax

-wrap

Description

This option wraps atoms back into the bounding box, using periodic boundary conditions. In other words, all atoms that are not in the simulation box (defined by the simulation box vectors H) are translated by one or several box vectors until they end up inside of the simulation box.

If some atoms have shells (in the sense of an ionic core-shell model) then the shells are translated by the same vector as their associated core, so that the vector between core and shell remains the same. That means that even if all the cores are replaced inside the box, some shells may end up outside of the box.

If a selection was defined (with the option -select) then only selected atoms are wrapped.

Default

By default atoms are not wrapped.

Examples

  • atomsk initial.cfg -wrap final.xyz

    This will read the file initial.cfg and wrap all atoms so that all coordinates are inside the simulation box. The result will be output into final.xyz.
