Nom : Dr Probe Cell File

Extension : .cel

Spécification : Site Web de Dr Probe

Programmes de visualisation : Dr Probe

Restrictions

Dr Probe est un logiciel de simulation d'images de microscopie haute résolution (HRTEM). Atomsk peut lire les fichiers au format CEL, avec les restrictions ci-dessous.

Pour écrire un fichier dans ce format il est recommandé de définir les taux d'occupation des sites et les paramètres vibrationnels en tant que propriétés auxiliaires de chaque atome. Ils peuvent être définis par exemple avec l'option -properties. Si ces propriétés ne sont pas définies, alors Atomsk écrira dans le fichier les valeurs par défaut 1 et 0, respectivement.

Notez que Atomsk ne réalise aucune conversion implicite des unités. Par défaut les fichiers CEL utilisent le nanomètre comme unité de longueur (et nm2 pour les vibrations thermiques), et Atomsk enregistre ces valeurs telles quelles sans les modifier. Il appartient à l'utilisateur de convertir explicitement ces valeurs s'il souhaite obtenir des Å (ou toute autre unité), par exemple avec l'option -unit. De même, si des données sont lues en Å (ou toute autre unité), il appartient à l'utilisateur de les convertir en nm avant d'écrire un fichier au format CEL. Les exemples ci-dessous illustrent ces points.

Examples

  • atomsk mon_systeme.xsf -unit A nm cel

    Ceci va lire le fichier mon_systeme.xsf (dans lequel les positions atomiques sont en Å), ensuite convertira les positions atomiques en nanomètres, puis écrira le résultat final dans mon_systeme.cel.

  • atomsk supercell.cel -unit nm A cfg

    Ceci va lire le fichier supercell.cel (où les positions atomiques sont en nanomètres), convertira les positions en Å, et écrira le résultat final dans supercell.cfg.

  • atomsk unitcell.cel -dupplicate 1 1 3 supercell.cel

    Ceci va lire le fichier unitcell.cel, dupliquera le systèmes trois fois suivant Z, et écrira le résultat final dans supercell.cel. Notez que puisque les fichiers initial et final utilisent les mêmes unités (càd des nanomètres) il n'y a pas besoin de les convertir.
