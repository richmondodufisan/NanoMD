Mode : Unfold

Syntaxe

atomsk --unfold <fichier> [<préfixe>] <formats> [options]

Description

Dans ce mode, Atomsk lit un fichier contenant plusieurs configurations atomiques, et écrit chaque configuration dans un fichier de sortie différent.

Les formats de fichier d'entrée pris en charge par ce mode sont :

  • HISTORY : fichiers produits par DL_POLY (voir les spécifications de DL_POLY) ; le pas de simulation du fichier HISTORY est utilisé pour nommer les fichiers de sortie.
  • Les fichiers de sortie personnalisés de LAMMPS (*.lmc) (voir les spécifications).
  • Les fichiers de sortie de Quantum Espresso PWscf (voir les spécifications).
  • Les fichiers de sortie OUTCAR de VASP (voir les spécifications).
  • Les fichiers XSF animés (voir les spécifications).
  • Les fichiers XYZ, au format par défaut ou au format étendu (voir les spécifications) ; les fichiers au format XYZ spécial ne sont pas reconnus dans ce mode.

Chaque configuration peut être écrite dans un ou plusieurs <formats>.

Si ce mode est employé avec une ou plusieurs options, celles-ci s'appliqueront à chaque configuration.

Pour l'opération inverse (réunir plusieurs configurations dans un seul fichier), voir le mode --gather.

Exemples

  • atomsk --unfold dump.lmc cfg

    Ceci va lire le fichier monfichier.lmc, et écrira chaque configuration dans un fichier CFG séparé : dump1.cfg, dump2.cfg, etc.

  • atomsk --unfold monfichier.xyz xsf

    Ceci va lire le fichier monfichier.xyz, et écrira chaque configuration dans un fichier XSF séparé : monfichier1.xsf, monfichier2.xsf, etc.

  • atomsk --unfold scf.out -unit B A xsf

    Ceci lira le fichier de sortie de Quantum Espresso PWscf scf.out, convertira les unités des vecteurs de boîte et des coordonnées des atomes de Bohrs vers Angströms, et écrira chaque configuration dans un fichier XSF : scf1.xsf, scf2.xsf, etc.

  • atomsk --unfold HISTORY xsf cfg

    Ceci lira le fichier HISTORY produit par DL_POLY, et écrira chaque configuration aux formats XSF et CFG : HISTORY1.xsf et HISTORY1.cfg, HISTORY2.xsf et HISTORY2.cfg, etc.

  • atomsk --unfold monfichier.xsf nouveau_ xsf cfg

    Cet exemple utilise un préfixe ("nouveau_"). Le fichier XSF animé monfichier.xsf sera lu, et chaque configuration sera convertie aux formats XSF et CFG : nouveau_1.xsf et nouveau_1.cfg, nouveau_2.xsf et nouveau_2.cfg, etc..
