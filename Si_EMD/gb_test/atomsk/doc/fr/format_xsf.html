Format : XSF

Nom : structure pour X-window Crystalline and Structure Densities (XCrySDen)

Extension : xsf

Spécification : documentation de XCrySDen

Programmes de visualisation : VESTA, VMD, XCrySDen

Restrictions

Atomsk peut lire les fichiers aux formats XSF contenant des structures périodiques ou moléculaires (voir la spécification). Pour les structures moléculaires, puisqu'aucune boîte n'est définie dans le fichier Atomsk créera automatiquement une boîte (ceci est identique à l'option -rebox). Pour définir une boîte manuellement il est possible d'utiliser l'option -properties. Si les forces pour chaque atome sont présentes dans le fichier XSF, Atomsk les enregistrera comme propriétés auxiliaires.

Lorsqu'il écrit un fichier au format XSF Atomsk utilise toujours une structure périodique. Si des propriétés auxiliaires existent, alors seules les forces (fx, fy, fz) seront écrites.

Si le fichier est un fichier XSF animé (contenant plusieurs images) alors seule la première image sera lue et convertie. Pour convertir toutes les images il est possible d'utiliser le mode one-in-all. Pour créer un fichier au format XSF animé, il est possible d'utiliser le mode all-in-one.

Exemples

  • atomsk systeme.xsf POSCAR

    Ceci lira le fichier XSF systeme.xsf, et le convertira en fichier POSCAR.

  • atomsk initial.cfg final.xsf

    Ceci lira le fichier initial.cfg, et le convertira en final.xsf.

  • atomsk --all-in-one mesfichiers.lst animation.xsf

    Ceci va lire les fichiers apparaissant dans la liste myfiles.lst, et écrira toutes les configurations dans le fichier XSF animé animation.xsf.

  • atomsk --one-in-all animation.xsf cfg

    Ceci va lire le fichier XSF animé animation.xsf, et écrira chaque configuration dans un fichier séparé au format CFG : animation_1.cfg, animation_2.cfg, etc.
