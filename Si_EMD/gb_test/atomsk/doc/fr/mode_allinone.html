Mode : Gather

Syntaxe

atomsk --gather <fichierliste> <fichiersortie> [<formats>] [options]

Description

Dans ce mode ("gather" veut dire "rassembler"), Atomsk lit plusieurs fichiers, et écrit toutes les configurations atomiques (ou "images") dans un seul fichier de sortie.

Les noms des fichiers d'entrée doivent apparaître dans le <fichierliste>, un fichier texte au même format que celui employé par le mode file list. Un tel fichier liste peut être facilement produit avec une commande du type "ls >maliste.lst".

Les formats de sortie supportés par ce mode sont :

  • Format XSF animé ("Animated XSF", seulement au format "variable-cell animated XSF", voir les spécifications du format XSF);
  • XYZ, soit au format XYZ standard soit au format XYZ étendu (voir les spécifications du format XYZ); notez que le format XYZ spécial n'est pas pris en charge par ce mode.

Pour le format XSF animé, si les forces sur les atomes existent dans le permier fichier du <fichierliste> alors ces forces seront écrites pour toutes les configurations dans le fichier XSF animé final. Si les forces sont manquantes dans un fichier suivant de la liste, alors les forces de cette configuration seront définies comme nulles dans le fichier XSF final. En revanche, si les forces n'apparaissent pas dans le premier fichier du <fichierliste>, alors aucune force ne sera écrite pour aucune configuration dans le fichier XSF animé final.

Lors de l'écriture vers le format XYZ standard, les propriétés auxiliaires sont ignorées et seules les positions atomiques sont écrites. Lors de l'écriture vers le format XYZ étendu, toutes les propriétés auxiliaires sont conservées et écrites dans le fichier final.

Plusieurs formats de sortie peuvent être spécifiés.

Si vous utilisez ce mode avec une ou plusieurs options, alors ces options seront appliquées à chaque configuration (chaque image) avant qu'elle ne soit écrite dans le fichier de sortie.

Pour l'opération inverse (séparer un fichier contenant plusieurs configurations dans plusieurs fichiers), voir le mode --unfold.

Exemples

  • atomsk --gather maliste.lst animation.xsf

    Ceci lira tous les fichiers dont les noms apparaissent dans maliste.lst, et écrira toutes les images dans le fichier XSF animé animation.xsf.

  • atomsk --gather maliste.lst images.xyz exyz xsf

    Ceci lira tous les fichiers dont les noms apparaissent dans maliste.lst, et écrira toutes les images dans les fichiers images.xyz (au format XYZ étendu) et images.xsf.
