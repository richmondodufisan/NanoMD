Nom : JEMS File

Programmes de visualisation : JEMS

Restrictions

JEMS est un logiciel de simulation d'images de microscopie haute résolution (HRTEM). Atomsk peut lire et écrire les fichiers au format JEMS, avec les restrictions ci-dessous.

Pour écrire un fichier dans ce format il est recommandé de définir les taux d'occupation des sites, les paramètres vibrationnels, et les coefficients d'absorption, en tant que propriétés auxiliaires de chaque atome. Ils peuvent être définis par exemple avec l'option -properties. Si ces propriétés ne sont pas définies, alors Atomsk écrira dans le fichier les valeurs par défaut 1, 0, et 0,03, respectivement.

Lors de la lecture d'un fichier au format JEMS, si un groupe d'espace est spécifié (dans une ligne commençant par le mot-clé "HMSymbol"), alors Atomsk applique immédiatement les opérations de symétrie associées à ce groupe.

Notez que Atomsk ne réalise aucune conversion implicite des unités. Par défaut les fichiers JEMS utilisent le nanomètre comme unité de longueur (et nm2 pour les vibrations thermiques), et Atomsk enregistre ces valeurs telles quelles sans les modifier. Il appartient à l'utilisateur de convertir explicitement ces valeurs s'il souhaite obtenir des Å (ou toute autre unité), par exemple avec l'option -unit. De même, si des données sont lues en Å (ou toute autre unité), il appartient à l'utilisateur de les convertir en nm avant d'écrire un fichier au format JEMS. Les exemples ci-dessous illustrent ces points.

Examples

  • atomsk mon_systeme.xsf -unit A nm JEMS

    Ceci va lire le fichier mon_systeme.xsf, convertira les unités de angströms en nanomètres, puis écrira le résultat final dans le fichier my_system.txt. Ce dernier pourra être ouvert dans JEMS.
