Nom : configuration pour MOLDY

Extension : non spécifié (Atomsk utilise .mol)

Spécification : Wiki de MOLDY

Restrictions

Atomsk peut lire les fichiers de configuration de MOLDY (par ex. "system.in" ou "system.out"). Comme indiqué dans la spécification, la seconde ligne doit contenir trois entiers correspondant au nombre de répliques du système dans chaque dimension. Atomsk utilise ces entiers pour effectuer cette réplication (ces entiers ont donc le même effet que l'option -duplicate).

Lorsqu'il écrit un fichier au format MOLDY, Atomsk utilise l'extension ".mol" après le nom de fichier. Les masses des atomes écrites dans le fichier sont celles des isotopes les plus communs, tels que définis par le National Institute of Standards and Technology (NIST).
