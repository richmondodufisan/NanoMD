Mode : Copy Properties

Syntax

atomsk --copy-properties <fichier1> <fichier2> [options] [<outputfile>] [formats]

Description

Ce mode copie les propriétés auxiliaires du <fichier1> vers le <fichier2>.

Ce mode copie toutes les propriétés auxiliaires du <fichier2> (s'il y en a), puis copie les propriétés du <fichier1>. Si une propriété porte le même nom dans les deux systèmes, alors les valeurs du <fichier2> sont écrasées par celles du <fichier1>. Sinon, une nouvelle colonne est créée pour enregistrer les données.

Le système final contient les positions des atomes et les propriétés auxiliaires du <fichier2>, ainsi que les propriétés auxiliaires copiées depuis le <fichier1>.

Les deux fichiers doivent avoir le même nombre d'atomes pour que les propriétés soient copiées pour tous les atomes. Sinon, si le <fichier1> et le <fichier2> contiennent des nombres d'atomes différents N1 et N2, alors les propriétés ne sont copiées que jusqu'au plus petit nombre d'atomes. Si N1 > N2, alors certaines valeurs des propriétés de <file1> ne pourront pas être copiées et seront perdues. Si N1 < N2, alors les atomes avec un indice supérieur à N1 auront une valeur égale à zéro pour les propriétés copiées.

Si vous utilisez ce mode avec une ou plusieurs options, alors elles sont appliquées au <fichier2> avant que les propriétés ne soient copiées.

Si une sélection a été définie (avec l'option -select), alors les propriétés auxiliaires ne sont copiées que pour les atomes sélectionnées. Les autres atomes (non sélectionnés) garderont les valeurs des propriétés qui étaient définies dans le <fichier2>, ou prendront la valeur zéro pour les propriétés qui n'existaient pas dans le <fichier2>.

Exemples

  • atomsk --copy-properties systeme1.cfg systeme2.cfg resultat.cfg

    Avec cette commande, Atomsk lit les deux fichiers systeme1.cfg et systeme2.cfg, et copie les propriétés auxiliaires. Le fichier de sortie final resultat.cfg contiendra les positions des atomes et les propriétés auxiliaires du systeme2.cfg, ainsi que les propriétés auxiliaires du systeme1.cfg. Si la même propriété existe dans les deux systèmes, seules les valeurs du systeme1.cfg apparaîtront dans le fichier final.
