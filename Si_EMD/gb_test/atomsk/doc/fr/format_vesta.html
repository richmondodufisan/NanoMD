Format : VESTA

Nom : fichier VESTA

Extension : vesta

Spécification : documentation de VESTA

Programmes de visualisation : VESTA

Restrictions

Atomsk peut lire les fichiers VESTA. Si des vecteurs sont définis (après le mot-clé "VECTR"), alors leurs composantes sont enregistrées comme propriétés auxiliaires sous les noms vecx, vecy, vecz. Si le fichier contient des opérations de symétrie (après le mot-clé "SYMOP"), alors elles sont appliquées immédiatement.

Lorsqu'il écrit un fichier VESTA, Atomsk ignore aussi les opérations de symétrie, c'est-à-dire que le groupe d'espace sera P1. Si, parmi les propriétés auxiliaires, il en existe trois portant le même préfixe suivi de x, y, z (par exemple les forces fx, fy, fz, ou bien les vitesses vx, vy, vz), alors elles seront écrites en tant que vecteurs dans le fichier VESTA, afin que ce dernier puisse les afficher comme vecteurs. Notez que si les composantes ont des valeurs très petites ou très élevées, alors les flèches seront quasiment invisibles ou bien démesurément grandes dans VESTA, ce qui peut être corrigé en redimensionnant les composantes grâce à l'option -unit. Si les forces sont définies en tant que propriétés auxiliaires (fx, fy, fz), alors elles sont écrites comme vecteur après la position de chaque atome. Sinon, si les vitesses des atomes sont définies alors elles sont utilisées comme vecteurs. Sinon les vecteurs seront nuls pour tous les atomes.

Exemples

  • atomsk initial.cfg final.vesta

    Ceci va lire le fichier initial.cfg, et le convertira en final.vesta.

  • atomsk CONTCAR vesta

    Ceci va lire le fichier VASP CONTCAR, et le convertira en CONTCAR.vesta.

  • atomsk file.vesta lammps

    Ceci va lire le fichier VESTA file.vesta, et le convertira en file.lmp.
