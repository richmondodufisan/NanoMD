Format : IMD

Nom : fichier de configuration pour le programme Institut für Theoretische und Angewandte Physik (ITAP) Molecular Dynamics

Extension : imd

Spécification: manuel d'IMD

Programme de visualisation : OVITO

Restrictions

Atomsk supporte seulement les fichiers IMD au format "atom configuration files" (et non les "distribution files"), et seulement au format ASCII format (c'est-à-dire ceux contenant "#F A" dans leur en-tête). Dans le format IMD le nombre total d'atomes n'est pas indiqué, par conséquent Atomsk doit parcourir le fichier une première fois pour compter les atomes, puis allouer la mémoire nécesaire, et enfin re-parcourir le fichier pour lire les positions des atomes et les charger en mémoire. Ceci rend la lecture de tels fichiers assez lente, surtout pour les fichies contenant un grand nombre d'atomes.

Dans le format IMD les espèces atomiques ne sont pas indiquées et sont remplacées par des "types" d'atomes (0, 1, 2...). Pour des raisons techniques ces "types" sont incrémentés par Atomsk (le type 0 devient type 1, le type 1 devient 2, etc.), et Atomsk les interprète comme des numéros atomiques (type 1=Hydrogène, 2=He, etc.). Ces "types" factices peuvent être remplacés grâce à l'option -substitute. Si les vitesses des atomes sont définies alors Atomsk les lit et les enregistre comme propriétés auxiliaires. Aucune autre propriété auxiliaire n'est pour l'instant supportée.

De manière similaire, lorsqu'il écrit un fichier IMD Atomsk remplace automatiquement les espèces des atomes par des "types" (0, 1, 2...). Alternativement ces "types" peuvent être définis grâce à l'option -properties. Si les vitesses des atomes sont définies (si par ex. elles ont été lues depuis un format qui le supporte) elle seront écrites dans le fichier IMD, sinon elles seront définies comme nulles.
