Format : XMD

Nom : XMD

Extension : xmd

Spécification : manuel de XMD

Programmes de visualisation : aucun

Restrictions

Atomsk peut lire les fichiers au format XMD contenant les dimensions de la boîte et les positions des atomes. De tels fichiers peuvent être produits par XMD avec les commandes WRITE BOX, WRITE PARTICLE, etc. (cf le manuel de XMD). Les types et les masses des atomes, si elles sont définies, sont enregistrées comme propriétés auxiliaires pour assurer leur conservation lors de la conversion vers d'autres formats de fichiers les supportant, comme les fichiers de données de LAMMPS, les fichiers de configuration IMD, ou bien les fichiers XMD. Notez que si le fichier XMD contient des instructions demandant à XMD de créer ou transformer un système atomique (avec des commandes comme DUP, FILL, SCREW, etc.), celles-ci ne sont pas exécutées par Atomsk.

Atomsk peut aussi écrire des fichiers XMD. Si les types et masses des atomes sont définies, elles sont utilisées pour générer le fichier XMD. XMD ne peut simuler que des boîtes orthogonales, autrement dit les vecteurs de boîte doivent être de la forme (H(1,1) 0 0), (0 H(2,2) 0) et (0 0 H(3,3)). Si ce n'est pas le cas, alors Atomsk affichera un message avertissant que tous les paramètres de boîte ne peuvent pas être écrits, et la géométrie du système final sera probablement incorrecte.

Le fichier XMD produit par Atomsk peut être lu dans un script XMD grâce à la commande READ fichier.xmd.

Exemples

  • atomsk systeme.cfg XMD

    Ceci va lire le fichier systeme.cfg, et le convertira en systeme.xmd.
