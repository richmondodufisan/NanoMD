Comportement du programme : verbosité

Syntaxe

-verbosity <n>

-v <n>

Description

Ceci permet de changer le niveau de verbosité du programme. L'entier <n> peut prendre les valeurs suivantes :

  • 0 : silencieux, aucun message n'est affiché à l'écran ni écrit dans un fichier, sauf les questions, alertes (warnings) et messages d'erreur.
  • 1 : les messages sont affichés à l'écran seulement.
  • 2 : les messages sont écrits dans le fichier journal uniquement. Les questions, alertes et erreurs apparaissent toujours à l'écran.
  • 3 : les messages sont affichés à l'écran et écrits dans le fichier journal.
  • 4 : niveau de déboguage, des messages supplémentaires sont écrits dans le fichier journal (mais pas à l'écran). Dans ce mode le programme peut être beaucoup plus lent, et le fichier journal peut être beaucoup plus volumineux. Ce mode ne devrait être utilisé que par des développeurs pour déboguer le programme, et seulement sur des systèmes comportant peu de particules.

Par défaut le fichier journal est atomsk.log mais il est possible d'en changer.

Lorsque le programme est exécuté en mode interactif le niveau de verbosité est automatiquement égal à 1.

Pour rendre ce changement permanent, vous pouvez éditer un fichier de configuration.

Défaut

Par défaut le niveau de verbosité est égal à 1.

Exemples

  • atomsk initial.gin initial.cfg -v 3

    Avec ceci le programme affichera ses messages à l'écran et les écrira dans le fichier journal atomsk.log.

  • atomsk initial.gin initial.cfg -v 0

    Un niveau de verbosité égal à 0 signifie que Atomsk n'affichera aucun message à l'écran, sauf les questions, alertes et messages d'erreurs.

  • atomsk initial.gin initial.cfg -v 0 &>/dev/null

    Même chose que l'exemple précédent, mais tous les messages (y compris les questions, alertes et erreurs) seront envoyés à /dev/null, ce qui signifie qu'ils ne seront pas affichés du tout. Notez que les questions, bien que non affichées à l'écran, attendront toujours une réponse, elles mettront donc Atomsk en pause jusqu'à ce qu'une réponse soit fournie. Cette solution devrait donc être réservée aux cas où vous êtes sûr que Atomsk ne posera pas de question.
