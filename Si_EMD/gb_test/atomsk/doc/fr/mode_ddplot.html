Mode : ddplot

Syntaxe

atomsk --ddplot <fichier1> <fichier2> [<fichiersortie>] [options]

Description

Dans ce mode, Atomsk lit deux fichiers d'entrée et écrit un fichier au format ddplot (.dd). Le <fichier1> doit correspondre au système idéal (sans dislocation) et le <fichier2> au système contenant une dislocation. Les deux fichiers DOIVENT contenir le même nombre de particules. Ils peuvent être dans des formats différents, tant que ces formats sont supportés.

Dans ce mode, tout <format> spécifié est ignoré, l'écriture n'étant possible qu'au format ddplot (.dd).

Si vous utillisez ce mode avec une ou plusieurs options, alors ces options seront appliquées à chacun des deux fichiers d'entrée avant que le fichier ddplot ne soit construit.

Notez qu'il existe une limite au nombre d'atomes que le programme ddplot peut afficher, et Atomsk affichera un message d'alerte si votre système dépasse ce nombre. Il est possible d'utiliser ce mode avec l'option -cut pour ne conserver que la partie du système intéressante à regarder.

Exemples

  • atomsk --ddplot initial.gin dislocation.xyz visuel.dd

    Ceci lira les deux fichiers initial.gin et dislocation.xyz, et créera le fichier ddplot visuel.dd.

  • atomsk --ddplot initial.gin dislocation.xyz -cut above 60 z -cut below 40 z visuel.dd

    Cet exemple utilise l'option -cut. Ici le programme lira les fichiers initial.gin et dislocations.xyz, et supprimera les atomes ayant une coordonnée suivant Z supérieure à 60 Å ou inférieure à 40 Å. Le résultat final sera écrit dans le fichier visual.dd.
